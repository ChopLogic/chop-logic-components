# ChopLogicButton

The `ChopLogicButton` component provides a flexible, accessible button with multiple visual styles and configurations. It supports text, icons, tooltips, and various interaction states.

## Usage

```tsx
import ChopLogicButton, { ButtonView, IconName } from 'chop-logic-components';

const ExampleForm = () => (
  <form>
    <ChopLogicButton
      view={ButtonView.Primary}
      text="Submit"
      icon={IconName.Check}
      onClick={() => console.log('Clicked!')}
    />
    <ChopLogicButton
      view={ButtonView.Icon}
      icon={IconName.Settings}
      label="Settings"
    />
  </form>
);
```

### Accessibility Notes

1. Automatic ARIA labeling
2. Keyboard focusable
3. Proper disabled state semantics
4. Tooltip integration
5. Screen reader support for icon buttons

### Best Practices

- Always provide labels for icon-only buttons
- Use primary buttons sparingly for the most important actions
- Combine icons with text when space allows
-  Consider button hierarchy in forms and dialogs
- Test keyboard navigation for all interactive elements
- Use appropriate button types (submit/reset/button) in forms
- Respect disabled states without removing interactive elements
