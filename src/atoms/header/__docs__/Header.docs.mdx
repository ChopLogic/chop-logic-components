# ChopLogicHeader

The `ChopLogicHeader` component provides a flexible heading element that can render as any HTML heading level (h1-h6) and optionally include an icon. It's designed to maintain consistent styling across all heading levels while providing semantic HTML structure.

## Usage

```tsx
import { ChopLogicHeader, IconName } from 'chop-logic-components';

const PageHeader = () => (
  <div>
    <ChopLogicHeader as="h1" icon={IconName.Home}>Welcome</ChopLogicHeader>
    <ChopLogicHeader as="h2">Section Title</ChopLogicHeader>
    <ChopLogicHeader as="h3" className="special-header" id="subsection">
      With Custom Class
    </ChopLogicHeader>
  </div>
);

export default PageHeader;
```

### When to Use

- For all heading elements in your application
- When you need semantic heading structure (h1-h6)
- When you want to maintain consistent heading styles across your application
- When you need headings with optional icons

### Accessibility Notes

1. __Heading Hierarchy__: Maintain a logical heading structure (h1 followed by h2, etc.) for proper document outline.
2. __Icon Decoration__: When using icons, ensure they're purely decorative or provide additional context via ARIA attributes if they convey meaning.
3. __Landmark Roles__: Consider wrapping standalone headers in `<ChopLogicHeader>` elements when appropriate.

### Best Practices

- Use only one h1 per page
- Maintain heading hierarchy (don't skip levels)
- Keep header text concise and descriptive
- Use icons sparingly and only when they add value
- Prefer semantic heading levels over visual appearance
