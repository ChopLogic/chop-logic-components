# Header

The `Header` component provides a flexible heading element that can render as any HTML heading level (h1-h6) and optionally include an icon. It's designed to maintain consistent styling across all heading levels while providing semantic HTML structure.

## Usage

```tsx
import { Header, IconName } from 'chop-logic-components';

const PageHeader = () => (
  <div>
    <Header as="h1" icon={IconName.Home}>
      Welcome
    </Header>
    <Header as="h2">Section Title</Header>
    <Header as="h3" className="special-header" id="subsection">
      With Custom Class
    </Header>
  </div>
);

export default PageHeader;
```

### When to Use

- For all heading elements in your application
- When you need semantic heading structure (h1-h6)
- When you want to maintain consistent heading styles across your application
- When you need headings with optional icons

### Accessibility Notes

1. **Heading Hierarchy**: Maintain a logical heading structure (h1 followed by h2, etc.) for proper document outline.
2. **Icon Decoration**: When using icons, ensure they're purely decorative or provide additional context via ARIA attributes if they convey meaning.
3. **Landmark Roles**: Consider wrapping standalone headers in `<Header>` elements when appropriate.

### Best Practices

- Use only one h1 per page
- Maintain heading hierarchy (don't skip levels)
- Keep header text concise and descriptive
- Use icons sparingly and only when they add value
- Prefer semantic heading levels over visual appearance
