# Tooltip

The `Tooltip` component provides contextual information when users interact with elements. It supports multiple trigger methods and positioning options, with accessible markup and flexible content.

## Usage

```tsx
import { Tooltip } from 'chop-logic-components';

const TooltipUsageExample = () => {
  return (
    <Tooltip tooltipContent="This is a tooltip" containerTag="span" visibleOn="hover" id="tooltip">
      <span>Hover over me</span>
    </Tooltip>
  );
};

export default TooltipUsageExample;
```

### When to Use

- To explain unfamiliar UI elements
- When you need to provide additional context
- For showing truncated content on hover
- To display helpful information without cluttering the interface
- When you need accessible tooltips that work with keyboard navigation

### Accessibility Notes

1. **ARIA Attributes**: Automatically manages aria-describedby and role="tooltip"
2. **Keyboard Navigation**: Works with focus trigger for keyboard users
3. **Focus Management**: Tooltip content remains accessible to screen readers
4. **Dismissal**: Click-triggered tooltips should be dismissible via ESC key
5. **Contrast**: Ensure sufficient color contrast for tooltip text/background

### Best Practices

- Keep tooltip text concise (1-2 sentences max)
- Use for supplemental information only
- Ensure tooltips don't obscure important content
- Test positioning at different screen sizes
- Prefer hover for most cases, click for complex content
- Avoid nesting interactive elements in hover-triggered tooltips
- Consider mobile touch interactions (hover may not work)
- Match tooltip appearance to your design system
